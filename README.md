# Reinforcement Learning
Reinforcement Learning (RL) is a type of machine learning where an **agent learns to make decisions by interacting with an environment**. The agent learns by receiving **rewards or penalties** based on its actions and aims to maximize long-term reward.

Unlike supervised learning, RL does not require labeled data. Instead, it learns through **trial and error**.


## Core Components of RL

An RL system is typically defined using the following components:

### 1. Agent
The learner or decision-maker that takes actions.

### 2. Environment
Everything the agent interacts with. It responds to actions and provides feedback.

### 3. State (S)
A representation of the current situation of the environment.

### 4. Action (A)
A set of possible moves the agent can take.

### 5. Reward (R)
A scalar feedback signal indicating how good or bad an action was.

### 6. Policy (π)
A strategy used by the agent to choose actions based on states.

### 7. Value Function
Estimates how good a state (or state-action pair) is in terms of future reward.

### 8. Model (optional)
A prediction of how the environment behaves (used in model-based RL).


## RL Interaction Loop

1. Agent observes state `S_t`
2. Agent selects action `A_t`
3. Environment returns:
   - Next state `S_{t+1}`
   - Reward `R_t`
4. Agent updates its policy/value based on experience

## On-Policy vs Off-Policy Learning

### On-Policy Learning
The agent learns **using the same policy it is currently following**.

- Learns from actions generated by the current policy
- More stable but less sample-efficient
- Must continuously interact with environment under the same policy

**Examples:**
- SARSA
- REINFORCE
- PPO (approximately on-policy)

**Intuition:**
> “I learn from my own current behavior.”

---

### Off-Policy Learning
The agent learns **from a different policy than the one it is currently using**.

- Can learn from past experiences or other agents
- More sample-efficient
- Can reuse replay buffers

**Examples:**
- Q-Learning
- DQN
- DDPG, SAC

**Intuition:**
> “I can learn from others or my past self.”

### Key Difference

| Aspect | On-Policy | Off-Policy |
|--------|----------|------------|
| Learning source | Current policy | Different / past policies |
| Sample efficiency | Lower | Higher |
| Stability | Higher | Can be less stable |
| Replay buffer | Not typical | Common |
| Example | SARSA, PPO | Q-learning, DQN |


## Types of Reinforcement Learning

### 1. Model-Free RL
The agent learns directly from experience without modeling the environment.
- Examples: Q-Learning, SARSA, Policy Gradients

### 2. Model-Based RL
The agent learns or is given a model of the environment and plans actions using it.


## Types of Learning Approaches

### 1. Value-Based
Learns value functions to decide actions.
- Example: Q-Learning, Deep Q Networks (DQN)

### 2. Policy-Based
Directly learns the policy function.
- Example: REINFORCE, PPO

### 3. Actor-Critic
Combines value-based and policy-based methods.
- Actor: chooses actions
- Critic: evaluates actions


## Key Concepts

- **Exploration vs Exploitation**
  - Exploration: trying new actions
  - Exploitation: using known best actions

- **Discount Factor (γ)**
  - Determines importance of future rewards

- **Episode**
  - A full sequence from start to terminal state

- **Return**
  - Total accumulated discounted reward


## Goal of RL

The goal is to learn an optimal policy π* that maximizes expected cumulative reward over time.
